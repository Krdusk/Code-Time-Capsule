# Code Time Capsule

Explore your Git history like never before. Code Time Capsule transforms your repository’s commit history, issues, and pull requests into an interactive timeline—letting you zoom, click, and ask questions about your project’s past.

Generated by **Natively AI** and **Google Gemma**.

## Overview

Code Time Capsule is a developer tool that visualises your repository’s complete history through an intuitive timeline interface. It ingests commit logs, issue trackers, pull request discussions, and CI/CD logs to provide contextual answers about any moment in your codebase’s evolution.

The project is designed to solve a common pain point: the difficulty of understanding why code was written a certain way, especially when joining a new team or revisiting old projects. By turning static Git logs into an interactive, queryable history, Code Time Capsule accelerates onboarding, debugging, and knowledge transfer.

---

## Features

**Interactive Timeline** – Visualise your repository’s entire commit history with zoomable, clickable event markers.

**Snapshot Viewing** – Click any event to see the repository state at that exact moment, including file structure and content.

**Natural Language Queries** – Ask questions about any point in history, such as “What was the team working on in March?” or “When was this bug introduced?”

**AI-Powered Answers** – Receive contextual, human-readable responses generated from commit messages, issues, pull requests, and CI logs.

**Exportable Reports** – Generate comprehensive summaries of any time period for documentation or onboarding.

**Dark and Light Mode** – Toggle between themes for comfortable viewing in any environment.

## Architecture

The application follows a modular architecture with clear separation of concerns:

**Frontend** – React components handle the user interface, including the timeline visualisation, snapshot panels, and question input. State management is handled via React contexts.

**Query Engine** – Processes user questions by filtering historical data and constructing context for the AI service.

**AI Service** – Integrates with Fireworks AI to generate answers based on commit history, issue discussions, and CI logs.

**Data Layer** – Parses and normalises data from GitHub APIs, JSON files, or CSV uploads.

## Getting Started

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build

# Preview production build
npm run preview
```

Test it here!: https://7xlhw8im7tdvxatstseakdpm5.nativelyai.app

### First Steps

1.  Open the Import modal from the sidebar or landing page.
2.  Paste a GitHub URL (for example, `https://github.com/facebook/react`) and click Import from GitHub.
3.  The Timeline, Snapshots, and Reports views populate instantly with live data.
4.  Click any event marker on the timeline to inspect it.
5.  Ask questions in the chat input at the bottom.

### Alternative: File Upload

You can also upload JSON or CSV files containing commits, issues, CI runs, or import graphs using the drop zone in the Import modal.

### Sample Queries

- “What was the team working on in March 2024?”
- “Show me the state of the authentication system before the refactor.”
- “How did our test coverage change over time?”
- “Who made the most changes to the payment module last year?”
- “When was this bug introduced?”
- “Summarise everything that happened in Q3.”
